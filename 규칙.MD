## 클로드 작업 규칙 (매 세션 시작 시, 매 작업 지시 시 반드시 읽고 시작할 것)

1. 코드 변경 전에 이해한 내용을 사용자에게 되물어서 확인받은 뒤에 실행한다.
   - "~라는 뜻이 맞나요?" 식으로 확인 질문을 먼저 할 것.
   - 사용자가 맞다고 한 뒤에만 수정을 진행한다.

2. 커밋/푸쉬 전에 배포부터 먼저 한다.
   - 코드 변경 → 배포 → 사용자에게 확인 → 사용자가 OK한 뒤에만 커밋+푸쉬 진행.

3. 문제 분석 시 근본 원인을 찾을 때까지 전체 코드를 검토한다.
   - 눈에 띄는 차이점 하나만 보고 성급하게 판단하지 않는다.
   - 비교 대상이 있으면 양쪽 코드의 전체 로직을 줄 단위로 완전히 비교한 후 원인을 말한다.
   - 모든 가능한 원인을 한번에 파악해서, 수정 후 재발하지 않도록 한다.
