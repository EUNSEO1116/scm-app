# 인천입고신청

## 개요
스프레드시트의 재고 입고 데이터를 자동으로 가져와 인천 입고신청용 엑셀을 생성하는 페이지.
기존 "입고신청"은 사용자가 엑셀을 업로드하면 가공하는 방식이고, 이 페이지는 스프레드시트에서 데이터를 추출하여 엑셀을 새로 만들어주는 방식.

- 경로: `/inventory/incheon`
- 컴포넌트: `src/pages/IncheonIncoming.jsx`
- 메뉴 위치: 재고관리 > 인천입고신청 (과잉재고 메뉴 대체)

---

## 데이터 소스

### 1. 재고 계산기 시트 (TSV)
- P열 (cols[15]) = `(1) 재고 입고` 수량
- 수량 > 0인 행만 추출
- 가져오는 값: 쿠팡바코드(cols[2]), 쿠팡 노출 상품명(cols[3]), 옵션명(cols[4])

### 2. 쿠팡바코드 시트 (CSV)
- 바코드(col5) → 입고센터(col11) 매핑

### 3. 특별 관리 상품 시트 (CSV)
- 바코드(col0) → 기타(1회성)(col8) 매핑
- 헤더 셀에 줄바꿈이 포함되어 있어, 바코드 형식(`S`+숫자)으로 실제 데이터 행만 필터링

---

## 기능

### 4. 상품개선 DB (improvement_items)
- `dbStoreGet('improvement_items')`로 조회
- 유형이 `CSV·VOC`이고 상태가 `시작전` 또는 `처리중`인 항목의 상품명 추출
- 상품명 키워드 매칭으로 VOC 표시

---

## 기능

### 1. 데이터 가져오기
- "데이터 가져오기" 버튼 클릭 시 3개 시트 + 상품개선 DB를 동시에 fetch
- 재고 입고 수량 > 0인 상품만 테이블에 표시

### 2. 엑셀 다운로드
- 파일명: `인천입고신청_YYYYMMDD.xlsx`
- 컬럼: 쿠팡바코드 | 상품명(노출상품명, 옵션명) | 재고 입고 수량 | 입고 센터 | 비고
- xlsx-js-style로 스타일 적용 (헤더 파란색, 테두리, 정렬)

### 3. 주문목록 업로드 (윙 발송 체크)
- `전계정_주문목록_날짜.xlsx` 파일 업로드
- 엑셀 열 매핑: D열(주문수량), E열(상품명), F열(옵션명)
- 상품명 + 옵션명이 정확히 일치하는 주문수량을 합산
- 비고 컬럼에 `N개 윙 발송` 표시 (파란색 배지)

### 4. 대기필요 등록
- 바코드를 복사붙여넣기로 입력 (줄바꿈, 쉼표, 탭 구분)
- 등록된 바코드가 입고 목록에 있으면 비고에 `대기필요` 표시 (주황색 배지)
- 개별 삭제 / 전체 삭제 가능

### 5. VOC 자동 표시
- 상품개선 DB에서 CSV·VOC 유형 + 시작전/처리중인 항목의 상품명을 키워드로 매칭
- 입고 목록의 상품명에 해당 키워드가 포함되면 비고에 `VOC` 표시 (빨간색 배지)

### 6. 특별관리 자동 표시
- 특별관리 시트의 기타(1회성)에 값이 있으면 비고에 `특별관리` 표시 (보라색 배지)

---

## 비고 컬럼 표시 규칙

네 가지 조건이 동시에 해당될 수 있으며, 모두 표시됨:

| 조건 | 표시 예시 | 배지 색상 |
|------|-----------|-----------|
| 주문목록 매칭 | `5개 윙 발송` | 파란색 (#e8f0fe) |
| VOC (상품개선 DB) | `VOC` | 빨간색 (#fce4ec) |
| 특별관리 (기타1회성) | `특별관리` | 보라색 (#f3e5f5) |
| 대기필요 등록 | `대기필요` | 주황색 (#fff3e0) |

엑셀 다운로드 시에는 쉼표로 합쳐서 텍스트로 출력 (예: `5개 윙 발송, VOC, 특별관리, 대기필요`)
