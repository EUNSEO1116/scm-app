# 발주신청 기능 버그 수정 이력

## 2026-04-10: CSV 파싱 버그 수정 (비고 쉼표 분리 + 발주번호 오류)

### 증상
- 비고란에 쉼표가 포함된 상품(예: `구성품 자꾸 틀림, 완제품 발송 요청,CN확인요망 + 설명서 동봉`)이 엑셀 복사/다운로드 시 쉼표 기준으로 다음 셀로 넘어감
- 일부 텍스트(예: `완제품 발송 요청`)가 아예 사라지는 현상
- 발주번호가 엉뚱한 코드로 생성됨 (브랜드/센터 컬럼이 밀려서 `AE-X` 등으로 빠짐)

### 원인
`parseCsvRows()` 함수의 이중 파싱 버그

1. `parseCsvRows`가 CSV 행을 분리하면서 **따옴표를 벗겨냄**
2. 따옴표 없는 문자열이 `parseCsvRow`에 전달됨
3. `parseCsvRow`는 쉼표를 전부 구분자로 처리 → 하나의 필드가 여러 컬럼으로 쪼개짐
4. 뒤의 컬럼(브랜드 cols[8], 센터 cols[11])이 밀려서 발주번호까지 틀어짐

**예시:**
```
CSV 원본: SKU,"구성품 자꾸 틀림, 완제품 발송 요청,CN확인요망",다음칼럼
parseCsvRows 통과 후: SKU,구성품 자꾸 틀림, 완제품 발송 요청,CN확인요망,다음칼럼
parseCsvRow 결과: [SKU, 구성품 자꾸 틀림, 완제품 발송 요청, CN확인요망, 다음칼럼] → 3개가 5개로 쪼개짐
```

### 수정 내용
`parseCsvRows`에서 따옴표를 보존하도록 수정 (`src/pages/OrderRequest.jsx`)

```javascript
// 수정 전: 따옴표 벗겨냄
if (ch === '"' && text[i + 1] === '"') { current += '"'; i++; }
else if (ch === '"') inQuotes = false;
else current += ch;
// ...
if (ch === '"') inQuotes = true;

// 수정 후: 따옴표 보존
if (ch === '"' && text[i + 1] === '"') { current += '""'; i++; }
else if (ch === '"') { inQuotes = false; current += '"'; }
else current += ch;
// ...
if (ch === '"') { inQuotes = true; current += '"'; }
```

추가로 쿠팡바코드 파싱도 `csv.split('\n')` → `parseCsvRows()` 로 통일하여 줄바꿈 포함 필드도 안전하게 처리하도록 변경.

### 영향 범위
- 비고란 쉼표/특수문자 포함 상품: 정상 표시
- 발주번호 생성: 브랜드/센터 컬럼 정상 인식으로 복구
- 엑셀 다운로드 및 TSV 복사 모두 정상
