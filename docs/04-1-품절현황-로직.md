## 4. 제외 로직 (목록에 표시 안 함)

### 4-1. 키워드 제외
상태에 아래 키워드 포함 시 완전 제외:
- `최종마감`
- `품질확인서`
- `마감대상`
- `덤핑`

### 4-2. 쿠팡바코드/재고계산기 미존재
- 업로드 데이터의 옵션ID가 쿠팡바코드 시트 또는 재고계산기 시트에 없으면 제외

### 4-3. 신규 상품 제외
- 신규 상품 중 쿠팡재고=0이고, 최근 7일 업로드 데이터에서 **한 번도 재고 > 0인 적 없으면** 제외 (아직 입고 자체가 안 된 신규라 품절이 아님)

### 4-4. 재고 있고 입고예정 있는 경우
- 재고 > 0이고 입고예정(`incoming`) 또는 입고(`ipgo`) > 0이면 → 위험 아님으로 판단, 목록에서 제외

### 4-5. 재고 수정된 항목
- `soldout_stock_corrections`에 등록된 항목은 품절/품절위기 판정에서 스킵 (유효 상품 집계에는 재고 있는 것으로 포함)

---

## 5. 위험 등급 판정

### 품절 (🔴)
- **조건:** 업로드 엑셀의 쿠팡재고(coupangStock) = 0
- 입고예정/입고 수량 있으면 → 사유: `입고예정 N개`, 입고예상일 = 내일
- 박스히어로 재고 있으면 → 사유: `BH재고 있음`
- 둘 다 없으면 → 사유: `재고 없음`

### 품절위기 (🟠)
- **조건:** 재고 > 0이고 입고예정 없으며, 아래 중 하나 해당:
  1. **예상 판매주 < 1주:** `weeksStockIncoming > 0 && weeksStockIncoming < 1`
     - 사유: `예상 X.X주`
  2. **3일 내 소진 예상:** `재고 / 3일평균판매 < 3`
     - 사유: `X.X일분`

### 판정 안 됨
- 품절도 품절위기도 아니면 → 목록에서 스킵

---

## 6. 3일 평균 판매량 계산

- 오늘 포함 최근 3일의 업로드 데이터(`soldout_analysis_YYYYMMDD`)를 DB에서 조회
- 옵션ID별 `salesQty` 합산 / 데이터 존재 일수 = 일평균
- 예상 판매주 = `(쿠팡재고 + 입고예정) / (일평균 × 7)`

---

## 7. 입고예상일 계산 (parseOrderBook)

발주장부의 CN 상태(cols[8])에 따라 입고예상일 산출:

| CN 상태 | 입고예상일 |
|---|---|
| `출고 완료` / `출고완료` | 중국출고일(cols[10]) + 3일 |
| `CN 창고도착` / `작업 대기` / `출고 대기` / `내륙운송` | 오늘 + 4일 |
| `업체발송대기` | 오늘 + 8일 |
| 상태 비어있음 | 계산 안 함 (null) |

### 윙 ON 판정 상태
아래 CN 상태 중 하나면 발주 상태를 `wing_on`으로 표시:
- `CN 창고도착`, `부분출고 대기`, `출고 완료`, `출고 대기`, `출고완료`

### 품절 + 입고예정 있는 경우
- 재고 = 0이지만 입고예정/입고 수량 > 0이면 → 입고예상일을 **내일**로 강제 설정

## 8. 품절률 계산

### 유효 품목 산정
전체 업로드 품목에서 아래를 제외:
- 키워드 제외 (`최종마감`, `품질확인서`, `마감대상`, `덤핑`)
- 쿠팡바코드/재고계산기 미존재
- 신규 상품 + 7일간 입고 기록 없음
- 재고 수정된 항목은 coupangStock=1로 간주 (품절 아님으로 처리)

### 계산식
```
품절률 = (재고 0인 유효 품목 수 / 전체 유효 품목 수) × 100
```

- 제외 품목(`soldout_analysis_exclude`)에 등록된 옵션ID는 분자·분모 모두에서 제외
- 소수점 둘째 자리까지 반올림
- 오늘 → 현재 제외 목록 실시간 반영 / 과거 날짜 → 캐시의 excludeSnapshot 사용

---

## 9. 정렬 기준

### 기본 정렬 (업데이트 시)
1차: **품절 먼저** (품절 > 품절위기)
2차: **3일 평균 판매량 높은 순** (판매량 많은 품절이 더 긴급)

### 테이블 컬럼 정렬
- 체크박스/제외 열을 제외한 **14개 컬럼** 클릭 정렬 가능
  - 상태, 등급, 옵션ID, 상품명, 옵션명, 입고예상, 쿠팡재고, 박스히어로, 총재고, 3일평균, 예상주, 발주현황, 품절일, 사유
- **품절(🔴)만 정렬 대상**, 품절위기(🟠)는 항상 하단 고정
- 기본 정렬: **상품명 오름차순**
- 클릭할 때마다 오름차순(▲) ↔ 내림차순(▼) 토글
- 비활성 컬럼에 `⇅` 아이콘, 활성 컬럼에 `▲`/`▼` 아이콘 표시
- 숫자 컬럼은 숫자 비교, 텍스트 컬럼은 한국어 로케일 비교
- null / 빈값 / `-`는 항상 맨 뒤로 밀림
- 달력에서 과거 날짜 조회 시에도 동일하게 동작

## 10. 재고 수정 기능

### 불일치 감지
- 업로드 엑셀의 쿠팡재고 = 0이지만 재고계산기의 쿠팡재고(calcStock) > 0인 경우 `mismatch` 플래그
- 테이블에서 쿠팡재고 셀 옆에 `수정 (N)` 버튼 표시

### 수정 동작
1. 클릭 시 `soldout_stock_corrections`에 해당 옵션ID 영구 저장
2. 캐시에서 해당 항목 제거 (품절 목록에서 사라짐)
3. `validItems`에서 해당 항목의 coupangStock을 1로 변경 (품절률 분모에서 재고 있는 것으로 처리)
4. 품절률 즉시 재계산 + 스냅샷 갱신

## 11. 품절 사유 관리

### 저장 구조
- `soldout_analysis_tracker` (DB): `{ optionId: { startDate, reason } }`
- 캐시의 `trackerSnapshot`에도 포함

### 사유 입력 방식
1. **개별 입력**: 사유 셀 클릭 → 텍스트 입력 → Enter 또는 저장 버튼
2. **일괄 입력**: 체크박스로 다중 선택 → "N개 사유 입력" 버튼 → 모달에서 일괄 사유 입력

### 사유 표시
- 오늘이면 현재 tracker 사용, 과거 날짜면 캐시의 trackerSnapshot 사용
- 품절위기 항목은 사유 대신 riskReason (예상 주, 일분 등) 표시

---

## 12. 품절 추적기

### 동작
1. 업데이트 시 품절(쿠팡재고=0) 옵션ID 목록 추출
2. 새로 품절된 항목은 `startDate = 오늘` 추가
3. 품절 해소된 항목(목록에 없는 기존 항목)은 tracker에서 삭제
4. `days` = 오늘 - startDate + 1 (최소 1일)

### 품절일 표시
- 7일 이상: 빨간색
- 3일 이상: 주황색
- 3일 미만: 기본색

## 13. 제외 품목 관리

### 저장소
`soldout_analysis_exclude` (DB): `[{ optionId, productName, optionName, status, barcode, excludedAt }]`

### 동작
- 테이블의 ✕ 버튼 클릭 시 제외 목록에 추가
- 제외된 항목은 투명도 0.45로 표시, "제외" 텍스트
- 품절률 계산 시 분자·분모 모두에서 제외
- 캐시의 `excludeSnapshot`에 반영 (오늘 + 보고 있는 과거 날짜 모두 갱신)

## 14. 신규 상품 재고 추적

### 저장소
`soldout_analysis_stock_tracker` (DB)

### 구조
```json
{
  "옵션ID": {
    "records": [{ "date": "YYYYMMDD", "stock": 0 }],
    "firstSeen": "YYYYMMDD"
  }
}
```

### 동작
- 업데이트 시 최근 7일 업로드 데이터 전체 스캔
- 신규 상태 상품의 쿠팡재고를 날짜별 기록
- `records`에 한 번이라도 `stock > 0`이 있으면 → 품절 판정 대상 (입고된 적 있으므로)
- 한 번도 `stock > 0`이 없으면 → 아직 미입고 신규 → 제외

## 15. 데이터 흐름 요약

```
[초기 로드]
DB 캐시(soldout_analysis_cached_오늘) → 즉시 표시

[업데이트 버튼]
1. 병렬 fetch: 쿠팡바코드 + 재고계산기 + 발주장부 + 업로드데이터 + 제외목록
2. 3일 평균 판매량 계산 (최근 3일 업로드 데이터)

5. 전체 유효상품 집계 + 품절/위기 판정

7. 추적기 갱신 (새 품절 추가, 해소된 품절 삭제)
8. 품절률 스냅샷 저장
9. 캐시 저장 (items, validItems, rate, trackerSnapshot, excludeSnapshot, correctionsSnapshot)

[달력 날짜 클릭]
DB 캐시(soldout_analysis_cached_해당날짜) → 표시 (없으면 "데이터 없음")
```
