## 3. 제외 로직 (목록에 표시 안 함)

### 3-1. 키워드 제외
상태에 아래 키워드 포함 시 완전 제외:
- `최종마감`
- `품질확인서`
- `마감대상`

### 3-2. 신규 상품 제외
- 신규 상품 중 7일간 재고 추적(`new_product_stock_tracker`)에서 **한 번도 재고 > 0인 적 없으면** 제외 (아직 입고 자체가 안 된 신규라 품절이 아님)

### 3-3. 재고 있고 입고예정 있는 경우
- 재고 > 0이고 입고예정(`incoming`) 또는 입고(`ipgo`) > 0이면 → 위험 아님으로 판단, 목록에서 제외

---

## 4. 위험 등급 판정

### 품절 (🔴)
- **조건:** 쿠팡재고(stock) = 0
- 입고예정/입고 수량 있으면 → 사유: `재고 없음 (입고예정 N개)`, 입고예상일 = 내일
- 박스히어로 재고 있으면 → 사유: `박스히어로 재고 있음`
- 둘 다 없으면 → 사유: `재고 없음`

### 품절위기 (🟠)
- **조건:** 재고 > 0이고 입고예정 없으며, 아래 중 하나 해당:
  1. **예상 판매주 < 1주:** `weeksStockIncoming > 0 && weeksStockIncoming < 1`
     - 사유: `예상 X.X주 (1주 미만)`
  2. **3일 내 소진 예상:** `재고 / 3일평균판매 < 3`
     - 사유: `재고 N개 / 일평균 X.X개 = X.X일분`

### 판정 안 됨
- 품절도 품절위기도 아니면 → 목록에서 스킵

## 5. 입고예상일 계산 (parseOrderBook)

발주장부의 CN 상태(cols[8])에 따라 입고예상일 산출:

| CN 상태 | 입고예상일 |
|---|---|
| `출고 완료` / `출고완료` | 중국출고일(cols[10]) + 3일 |
| `CN 창고도착` / `작업 대기` / `출고 대기` / `내륙운송` | 오늘 + 4일 |
| `업체발송대기` | 오늘 + 8일 |
| 상태 비어있음 | 계산 안 함 (null) |

### 윙 ON 판정 상태
아래 CN 상태 중 하나면 발주 상태를 `wing_on`으로 표시:
- `CN 창고도착`, `부분출고 대기`, `출고 완료`, `출고 대기`, `출고완료`

### 품절 + 입고예정 있는 경우
- 재고 = 0이지만 입고예정/입고 수량 > 0이면 → 입고예상일을 **내일**로 강제 설정

## 6. 품절률 계산

### 유효 품목 산정
전체 품목에서 아래를 제외:
- 키워드 제외 (`최종마감`, `품질확인서`, `마감대상`)
- 품절 제외 목록 (`soldout_exclude_items`)에 등록되고 종료일 미경과
- 신규/NEW 상태 + 입고예정 > 0

### 계산식
```
품절률 = (재고 0인 유효 품목 수 / 전체 유효 품목 수) × 100
```

소수점 둘째 자리까지 반올림.

---

## 7. 정렬 기준

1차: **품절 먼저** (품절 > 품절위기)
2차: **3일 평균 판매량 높은 순** (판매량 많은 품절이 더 긴급)

---

## 8. 품절 사유 관리

### 저장 구조
- `soldout_reasons_v2` (localStorage): `{ barcode: { reason, date } }`
- `soldout_history` (localStorage): `{ barcode: [{ reason, date, productName, optionName }] }`

### 사유 입력 흐름
1. 다중 선택 (체크박스)
2. 사유 텍스트 입력
3. 저장 시:
   - `soldout_reasons_v2`에 현재 사유 저장
   - `soldout_history`에 이력 누적 추가
   - 스프레드시트 동기화 (GET 배치 전송, 3개씩)
   - `markLocalSave()`로 보호 기간 등록

### 사유 복원
- `soldout_reasons_v2`에 사유 없는 바코드 → `soldout_history`에서 최신 사유 자동 복원

### 사유 삭제
- 개별 삭제 시 localStorage + 스프레드시트 모두 삭제

---

## 9. 품절 재진입 처리

### 동작 원리
1. 이전 품절 바코드 목록을 `soldout_prev_barcodes` (localStorage)에 저장
2. 다음 데이터 조회 시 비교:
   - 이전에 품절 아니었다가 **다시 품절**된 항목 → 앱 사유 삭제 (다시 입력 유도)
   - **스프레드시트 기록은 유지** (삭제하지 않음)
3. `markLocalSave()`로 보호된 바코드는 재진입 삭제 대상에서 제외

### 목적
- 품절 해소 후 재발생 시 이전 사유가 아닌 새 사유를 입력받기 위함

## 10. 신규 상품 재고 추적

### 저장소
`new_product_stock_tracker` (localStorage)

### 구조
```json
{
  "바코드": {
    "records": [{ "date": "YYYY-MM-DD", "stock": 0 }],
    "firstSeen": "YYYY-MM-DD"
  }
}
```

### 동작
- 페이지 조회 시 신규 상태 상품의 재고(G열)를 매일 기록
- 7일 초과 기록은 자동 제거
- 30일 이상 기록 없는 상품은 트래커에서 삭제
- `hadStockBefore()`: 한 번이라도 재고 > 0이었는지 확인 → 품절 판정에 사용
