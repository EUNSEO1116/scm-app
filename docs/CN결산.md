# CN 결산 기능

## 개요
CN(중국) 주간 거래내역 엑셀을 업로드하여 브랜드별 발주 비용을 결산하는 기능.

## 페이지 구성

### 1. 거래 데이터 업로드 (`/cn-settlement/upload`)
- 플로우 내보내기 엑셀(.xlsx) 업로드
- 환율(CNY→KRW) 수동 입력
- 필터: 거래유형 `오더 지불`, `환불`, `추가비용` + 상태 `승인`만
- 저장 컬럼: 날짜, 거래유형, 입/출금(위안화), 발주번호
- 업로드 이력 테이블 (삭제 가능)
- 카드 제목: 엑셀 내 날짜 범위 자동 생성 (예: `5/29~6/5 발주내역`)

### 2. 결산 기록 (`/cn-settlement/history`)
- 업로드된 거래 데이터를 브랜드별로 집계하여 표시
- 브랜드 매핑 설정 UI (좌측 상단, DB 저장)
- 각 카드: 브랜드별 기존/신규 발주금액(KRW), 추가비용, 합계
- 개별 거래 내역 접이식 표시
- 카드 삭제 → DB 반영

### 3. 결산 대시보드 (`/cn-settlement/dashboard`)
- 월별 그룹핑, 현재 월 자동 펼침 + "이번 달" 뱃지
- 표 형식: 주차 | 기간 | 브랜드별 기존/신규 | 합계
- 월 합계 행, 전체 펼치기/접기
- 검색 기능 (월, 날짜, 브랜드명)

## 브랜드 매핑

발주번호 `AE-{코드}-...`에서 두 번째 세그먼트로 브랜드 판별:

| 코드 | 브랜드 |
|------|--------|
| SH | 생활기준 |
| S | 어리플 |
| I, I2, M | 일상포인트 |
| HM, HM2 | 하루모음 |
| B | 로즈바운드 |
| O | 원데이홈 |
| T | 토글리 |
| L | 리빙스타일 |
| P | 펄빈 |
| E | 타플벨 |
| BS | 제외 (계산 안 함) |

- `-NEW` 포함 → 신규, 그 외 → 기존
- `-JJ`, `-JJJ`, `-Z` 등 중간 세그먼트 무시
- 브랜드 매핑은 결산 기록 페이지에서 UI로 추가/수정/삭제 가능 (DB 저장)

## 계산 로직

- **오더 지불**: 위안화 × 환율 = 발주금액(KRW)
- **환불**: 해당 브랜드 발주금액에서 차감
- **추가비용**: 발주금액과 별도 집계
- **환불 이월**: 발주 없이 환불만 있으면 마이너스 표시 + 다음 주 업로드에서 자동 차감, "전주 환불 반영" 문구 표시

## DB 저장소

| 키 | 용도 |
|----|------|
| `cn_settlement_data` | 업로드된 거래 데이터 (배열) |
| `cn_settlement_rate` | 마지막 입력 환율 |
| `cn_brand_mapping` | 브랜드 코드→이름 매핑 |
