# 품절 분석 기능

## 메뉴 구조

품절 분석 (상위 메뉴)
├── (NEW)품절 현황 → /soldout-analysis
├── (NEW)품절기록 → /soldout-analysis/history (준비 중)
├── (NEW)제외품목관리 → /soldout-analysis/exclude
├── (NEW)월 품절률 → /soldout-analysis/rate
└── 데이터 업로드 → /soldout-analysis/upload

## 데이터 업로드

- **소스**: 빅셀 로켓그로스 매출분석 엑셀 (.xlsx)
- **추출 컬럼**: 옵션ID, 상품명, 옵션명, 쿠팡재고, 판매수량
- **저장**: DB `soldout_analysis_YYYYMMDD` (날짜별)
- **빈도**: 하루 1회
- **용량**: 약 0.42MB/일 (3,390개 품목 기준), 1년 ~153MB

## (NEW)품절 현황

### 데이터 소스
| 항목 | 소스 |
|------|------|
| 쿠팡재고 | 업로드 엑셀 |
| 3일 평균 판매량 | 업로드 데이터 최근 3일치 자체 계산 |
| 예상판매주 | 자체 계산: (쿠팡재고+입고예정) / (3일평균×7) |
| 박스히어로, 입고예정, 총재고 | 재고 계산기 시트 (스프레드시트) |
| 상태(효자/마감대상 등) | 쿠팡바코드 시트 (B열=옵션ID, J열=상태) |
| 발주현황, 입고예상일 | 발주장부 시트 (스프레드시트) |

### 품절 계산 로직 (기존 품절관리와 동일)
1. **제외**: 최종마감, 품질확인서, 마감대상 상태
2. **재고계산기 시트에 없는 품목** 제외
3. **신규 상품**: 7일간 한 번도 재고 > 0이 된 적 없으면 품절 아님 (DB `soldout_analysis_stock_tracker`에서 추적)
4. **품절**: 쿠팡재고 0
5. **품절위기**: 예상판매주 < 1주 또는 재고/3일평균 < 3일
6. **입고예정 있으면**: 재고 > 0인 경우 목록에서 제외

### 운영 흐름
1. **데이터 업로드** (하루 1회) → 엑셀에서 재고/판매량 추출 → DB 저장
2. **업데이트 버튼** (하루 1회) → 스프레드시트 3개 + DB 데이터 연동 → 분석 결과 DB 캐시 저장
3. 이후 접속 시 → DB 캐시만 로드 (빠름), 업데이트 버튼 안 뜸
4. 다음 날 → 새 날짜라 캐시 없음 → 업데이트 버튼 다시 나타남

### 캐시 구조 (`soldout_analysis_cached_YYYYMMDD`)
```json
{
  "items": [...],           // 품절/위기 품목 배열
  "updatedAt": "ISO날짜",
  "rate": 5.2,              // 품절률
  "trackerSnapshot": {...}, // 그 당시 사유/품절일수
  "validItems": [...],      // 전체 유효 상품 (품절률 분모용)
  "excludeSnapshot": [...]  // 그 당시 제외 옵션ID 목록
}
```

### 품절률 계산
- **분모**: 전체 유효 상품 (최종마감/신규 필터 후) - 제외 품목
- **분자**: 그 중 쿠팡재고 0인 것
- 제외 버튼 누르면 **실시간 반영**
- 업데이트 시 스냅샷으로 DB 저장 → 월 품절률에 기록

### 달력 기능
- 필터바 우측 달력 아이콘 → 팝업 달력
- 날짜 클릭 → 해당일 캐시 로드 → 전체 대시보드 전환
- 과거 날짜: 그 당시 사유(trackerSnapshot), 제외 상태(excludeSnapshot) 표시

### 사유 입력
- 개별: 사유 칸 클릭 → 입력 → Enter/저장 버튼
- 일괄: 체크박스 선택 → "N개 사유 입력" 버튼 → 모달에서 일괄 적용
- 저장된 사유: 보라색 뱃지로 표시 (클릭 시 편집)
- DB: `soldout_analysis_tracker` (optionId별 startDate, reason)

### 품절일수
- 재고 0 시작일 기록 → 연속 일수 자동 계산
- 재고 1 이상 되면 tracker에서 삭제 → 대시보드에서 해제
- 재품절 시 이전 사유 초기화 (빈 사유로 시작)

## (NEW)제외품목관리

- 품절 현황에서 제외 버튼 → 여기에 기록 (optionId, 상품명, 옵션명, 등급, 바코드, 제외일시)
- 해제 버튼으로 품절률 계산에 다시 포함
- DB: `soldout_analysis_exclude` (배열)

## (NEW)월 품절률

- 업데이트 시 일별 스냅샷 자동 저장 → DB: `soldout_analysis_rate_snapshots`
- **월별 / 분기별 / 반기별** 뷰 전환
- 연도 드롭다운 선택
- 기간별 카드: 품절률 %, 상태(위험/주의/양호), 일평균 품절/전체, 기록일수
- 일별 상세 테이블 (최근 30일)

## DB 키 목록
| 키 | 용도 |
|----|------|
| `soldout_analysis_YYYYMMDD` | 날짜별 업로드 데이터 (엑셀 추출) |
| `soldout_analysis_cached_YYYYMMDD` | 날짜별 분석 결과 캐시 |
| `soldout_analysis_tracker` | 품절 추적기 (사유/시작일) |
| `soldout_analysis_exclude` | 제외 품목 목록 |
| `soldout_analysis_stock_tracker` | 신규 상품 7일 재고 추적 |
| `soldout_analysis_rate_snapshots` | 일별 품절률 스냅샷 |

## 기존 품절관리와의 차이
| 항목 | 기존 품절관리 | (NEW)품절 분석 |
|------|-------------|---------------|
| 쿠팡재고 | 스프레드시트 실시간 | 업로드 엑셀 (하루 1회) |
| 3일 평균 | 스프레드시트 cols[26] | 업로드 데이터 3일치 자체 계산 |
| 예상판매주 | 스프레드시트 cols[21] | 자체 계산 |
| 상태 | 재고계산기+데이터입력 시트 | 쿠팡바코드 시트 |
| 로딩 속도 | 매번 스프레드시트 fetch | DB 캐시 (빠름) |
| 품절률 기록 | localStorage + DB | DB만 (날짜별 스냅샷) |
