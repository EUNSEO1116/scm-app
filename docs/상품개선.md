# 상품개선 기능

## 개요
이슈관리 하위 메뉴로 상품 개선 요청 및 진행 상황을 관리하는 페이지.

- 경로: `/issue/improvement`
- 파일: `src/pages/ProductImprovement.jsx`

## 메뉴 구조
사이드바 이슈관리 하위 메뉴:
- **특별관리** (`/issue`) - 기존 특별 관리 상품 (IssueManagement.jsx)
- **상품개선** (`/issue/improvement`) - 상품개선 관리 (ProductImprovement.jsx)

## 요약 카드 (6개)
| 카드 | 포함 유형 | 상태 |
|---|---|---|
| 전체 | 모든 유형 | 전체 |
| 수배 대기 | 재수배 + 업체문제 | 시작전 |
| 수배 진행중 | 재수배 + 업체문제 | 처리중 |
| 개선 대기 | 상품문제 + CSV·VOC | 시작전 |
| 개선 진행중 | 상품문제 + CSV·VOC | 처리중 |
| 완료 | 모든 유형 | 완료 |

카드 클릭 시 해당 항목만 필터링, 다시 클릭하면 해제.

## 유형
- 재등록
- 재수배
- 업체문제
- 상품문제
- CSV·VOC

## 상태
- 시작전 (회색)
- 처리중 (주황)
- 완료 (초록) - 완료 선택 시 종료일 자동 입력

## 항목 등록 폼
- **유형** (필수): 드롭다운 선택
- **상태**: 기본 '시작전'
- **발생일**: 기본 오늘 날짜
- **종료일**: 선택 입력
- **상품명** (필수): 특별관리 품목에서 자동완성 검색 또는 직접 입력
- **바코드**: 자동완성 선택 시 자동 입력, 직접 입력 가능
- **초기 이슈 내용**: 텍스트 입력 → 타임라인 첫 번째 항목으로 저장
- **참고 URL**: 최대 3개 입력
- **첨부 사진**: 최대 5장, 파일 선택 또는 Ctrl+V 붙여넣기

## 카드 리스트
- 기본 **접힌 상태**: 상태, 유형, 상품명, 발생일, 기록 건수, 사진/URL 아이콘만 표시
- 클릭하면 **펼침**: 바코드, 종료일, URL 링크, 타임라인, 사진 버튼, 삭제 버튼

## 타임라인 (진행상황)
- 시간순으로 이슈/진행상황 기록
- Enter 또는 '추가' 버튼으로 새 항목 추가
- 각 항목 개별 삭제 가능
- 최신 항목은 파란 점, 이전 항목은 회색 점

## 첨부 자료
- 항목당 최대 5장
- 파일 선택 또는 Ctrl+V 클립보드 붙여넣기
- 800px 자동 리사이즈 (JPEG 70% 품질)

## 다운로드
- **엑셀 다운로드**: 전체 항목을 xlsx 파일로 다운로드 (번호, 상태, 유형, 상품명, 바코드, 발생일, 종료일, 이슈/진행상황, 첨부파일수)
- **사진 ZIP 다운로드**: 상품명 폴더별로 사진을 zip 압축 다운로드

## 필터
- 검색: 상품명, 바코드, 이슈 내용
- 유형 필터: 드롭다운
- 상태 필터: 드롭다운
- 초기화 버튼: 모든 필터 해제

## 데이터 저장
- **주 저장소**: Vultr SQLite DB (`improvement_items`, `improvement_images`)
- localStorage와 DB 이중 저장 (병합)
- `dbStoreSet` / `dbStoreGet` 으로 저장/조회
- DB 등록 완료 (2026-04-13): VALID_STORES + 테이블 생성 완료

## 특별관리 시트 알림

상품개선에 등록한 바코드가 특별관리 스프레드시트에 새로 나타나면 알림을 띄운다.

### 동작 흐름

1. **항목 등록 시** → 바코드가 시트에 없으면 감시 목록(`imp_watch_barcodes`)에 저장
2. **페이지 로드 시** → 감시 목록 바코드가 시트에 나타나면 알림 표시 (영구 유지)
3. **적용완료 클릭** → 알림 + 감시 목록에서 영구 제거
4. **이미 시트에 있는 바코드** → 감시 대상 아님, 알림 안 뜸

### localStorage 키

| 키 | 용도 |
|---|------|
| `imp_watch_barcodes` | 감시 목록 (시트 미등록 바코드) |
| `imp_pending_sync_alerts` | 영구 알림 목록 (시트에 나타난 것) |

### 알림 UI
- 이슈관리 알림과 동일한 CSS (주황색 배경 `#fff3e0`, 테두리 `#ffb74d`)
- 상단에 표시, "적용 완료" 버튼으로 수동 해제

## 발주장부 확인필요 연동

- 상품개선에서 완료되지 않은(시작전/처리중) 항목의 바코드가 발주장부에 있으면 확인필요로 표시
- 사유 형식: `상품개선: 유형(상태)` (예: `상품개선: 업체문제(처리중)`)
- `OrderBook.jsx`의 `computeCheckFlags` 함수에서 `improvement_items` localStorage 데이터 참조

## 발주신청 재수배 알림 모달

발주신청에서 전체 복사/그룹 복사 시, 상품개선의 재수배 항목과 매칭되면 모달에 함께 표시.

### 조건
- 상품개선 유형 = **재수배**
- 상태 = **시작전** 또는 **처리중**
- 상품개선 상품명이 발주신청 한글옵션명에 **키워드로 포함**될 때

### 모달 표시
- 기존 이슈관리 사진 모달과 동일한 화살표 네비게이션 (◀ ▶)
- 보라색 배지 `상품개선 · 재수배`, 헤더 `🔄 상품개선 재수배 알림`
- 표시 내용: 상품명, 바코드, 첨부 사진
- 이슈관리 사진 → 상품개선 재수배 순서로 표시

### 데이터 소스
- `improvement_items` (DB + localStorage 폴백)
- `improvement_images` (DB + localStorage 폴백)

## 상품명 자동완성
- 특별관리 탭의 품목 목록(Google Sheet '특별 관리 상품')에서 검색
- 상품명 또는 바코드로 검색 가능
- 선택 시 바코드 자동 입력
- 직접 입력도 가능
