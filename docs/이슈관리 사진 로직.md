## 저장 구조

- `issue_img_data` — 모든 상품의 이미지를 하나의 저장소에 저장. `{ [barcode]: [base64_img, ...] }` 형태
- `issue_img_counts` — 바코드별 이미지 개수. `{ [barcode]: count }` 형태
- 이미지는 Base64 JPEG, 최대 800px 리사이즈, 상품당 최대 3장

## 관련 파일

- `src/pages/IssueManagement.jsx` — `saveImages`, `openImgModal`, `handleImgAdd`, `handleImgPaste`, `handleImgDelete`
- `src/pages/OrderRequest.jsx` — `showPhotosAfterCopy` (복사 후 사진 모달 표시), 화살표 네비게이션
- `src/utils/dbApi.js` — `dbStoreGet`, `dbStoreSet` (Vultr 프록시 경유)
- `api/proxy.js` — Vercel → Vultr(158.247.239.161:3100) 프록시

## Vultr 서버 (server.js)

- `VALID_STORES` 배열에 `'issue_img_data'`, `'issue_img_counts'` 추가됨
- `!name.startsWith("issue_img_") &&` 조건도 POST/GET 핸들러에 추가됨
- SQLite 테이블은 VALID_STORES에 추가해도 자동 생성 안 됨 → `sqlite3 /root/scm-api/scm.db "CREATE TABLE IF NOT EXISTS ..."` 명령으로 수동 생성 필요

## 주의사항

- 새 store 이름 추가 시: VALID_STORES에 추가 + sqlite3로 테이블 수동 생성 + pm2 restart all
- 테이블 스키마: `(id INTEGER PRIMARY KEY AUTOINCREMENT, data TEXT)`
