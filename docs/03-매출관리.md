# 매출관리

## 개요
일일 판매량 추적, 판매 급증 감지, 추세 분석 기능.

---

## 1. 매출관리 (Sales.jsx)
**경로:** `/sales`

### 데이터 소스
- Google Sheets '일일 판매량' 시트 (6일치 판매 데이터)
- Google Sheets '재고 계산기' 시트 (상태 정보)
- '주의 품목' 시트 (sheetSync.js 연동)

### 판매 급증 감지 (신규 판매 급증)
상단 카드 영역에 급증 항목 표시

**감지 조건:**
- 상태 = '신규' 이면서
- 당일(1일전) 판매 ≥ 4개, 그리고
  - 당일 ≥ 이전 5일 평균 × 2 또는
  - 당일 ≥ 이전 5일 최대값 + 3

**표시:** 미니 6일 바 차트 + 이전 평균 → 당일 판매량, 급증 차이

### 메인 테이블 컬럼
| 컬럼 | 설명 |
|---|---|
| 상태 | 신규, 효자, 기준미달, 최종마감 |
| 바코드 | 상품 바코드 |
| 상품명 | 클릭 시 주의 품목 토글 |
| 옵션명 | — |
| 총재고 | 현재 총 재고 |
| 6일전~1일전 | 일별 판매량 (녹색 그라데이션) |
| 합계 | 6일 판매 합계 |
| 리뷰갯수(6일) | 6일간 리뷰 수 |
| 추세 | 추세 지표 |

### 추세 지표
| 표시 | 조건 |
|---|---|
| ↑↑ {배율}x | 급증 (≥ 2배) |
| ↑ {배율}x | 증가 (1.3~2배) |
| → {배율}x | 유지 (0.7~1.3배) |
| ↓ {배율}x | 감소 (0.5~0.7배) |
| ↓ 0 | 판매 중단 |

### 주의 품목 (⚠️)
- 상품명 클릭 → 주의 플래그 토글
- Vultr DB `caution_items` 테이블에 저장 (barcode, product_name, option_name)
- API 경로: `POST /api/caution` (추가), `GET /api/caution` (조회), `DELETE /api/caution/:barcode` (해제)
- 프론트: `dbApi.js` → `sheetSync.js` → `Sales.jsx` (cautionSet State로 관리)
- 주황색 하이라이트 표시
- 필터 옵션: "⚠️ 주의 품목"

**버그 수정 이력 (2026-04-10):**
- 주의 품목 해제(DELETE) API 경로에 `/api/`가 중복되어 해제가 DB에 반영되지 않던 버그 수정
- `dbRemoveCaution` URL: `${API_BASE}/api/caution/...` → `${API_BASE}/caution/...`

### 셀 색상
- 판매량 셀: 녹색 그라데이션 (판매량 높을수록 진한 녹색)
